# Antigravity Task: Build Missing Frontend Pages

The admin-vite React app at `ui/admin-vite/` has several backend controllers with NO frontend pages. Build all of them following the existing page patterns (look at Warehouse, Role, or User pages for reference).

## Pattern Reference

Each page follows this structure:
```
src/features/{module}/{feature}/
‚îú‚îÄ‚îÄ pages/{Feature}Page.tsx       ‚Äî main page component
‚îú‚îÄ‚îÄ components/                    ‚Äî table, modal, form components
‚îú‚îÄ‚îÄ api/{feature}.api.ts           ‚Äî API calls (if needed)
‚îî‚îÄ‚îÄ types/index.ts                 ‚Äî TypeScript types (if needed)
```

Add routes in `src/app/router/index.tsx` and route constants in `src/constants/routes.ts`.

Use existing UI components from `src/components/ui/`: Table, Button, Modal, Input, Select, Loader, DataTable, Tabs, Toast, etc.

API calls go through `apiClient` from `src/lib/axios.ts` ‚Äî all endpoints are already defined in `src/constants/api.ts` where possible.

## Task 1: Finance Pages

### 1a: Chart of Accounts Page

**Route:** `/finance/chart-of-accounts`
**Backend:** `ChartOfAccountsController` in Finance.API (port 5008)
**API endpoints:**
- POST `/finance/chartofaccounts/get-grid-data` ‚Äî list COA as tree
- POST `/finance/chartofaccounts/save` ‚Äî create/edit account
- GET `/finance/chartofaccounts/get/{id}` ‚Äî get single account
- POST `/finance/chartofaccounts/delete` ‚Äî delete account

**Features:**
- Tree view showing account hierarchy (parent_account_id ‚Üí level structure)
- Columns: account_code, account_name, account_type, normal_balance, level, is_transactional
- Modal for create/edit with fields: account_code, account_name, account_type (dropdown: Asset/Liability/Equity/Income/Expense), parent_account (tree dropdown), normal_balance (Dr/Cr), is_transactional (checkbox)
- Follow same pattern as Category tree view (see `src/features/scm/category/`)

### 1b: Vouchers Page

**Route:** `/finance/vouchers`
**Backend:** `VouchersController` in Finance.API
**API endpoints:**
- POST `/finance/vouchers/get-grid-data` ‚Äî list vouchers
- GET `/finance/vouchers/get/{id}` ‚Äî get voucher with details
- POST `/finance/vouchers/create` ‚Äî create voucher (double-entry)
- POST `/finance/vouchers/reverse/{id}` ‚Äî reverse/cancel voucher

**Features:**
- DataTable with columns: voucher_id, voucher_type (PV/RV/JV/CV), voucher_date, company_id, is_cancelled
- Create voucher modal with:
  - Header: voucher_type (dropdown), voucher_date (date picker), company_id
  - Detail lines table: account_id (dropdown from COA), debit_amount, credit_amount, ref_type, ref_id, narration
  - Auto-validate Dr = Cr before save
- Detail view page showing voucher with all lines

## Task 2: DSR Pages

All under `src/features/sales/dsr/` ‚Äî follow the existing `src/features/sales/visit-plans/` pattern.

### 2a: DSR Attendance Page

**Route:** `/sales/dsr/attendance`
**Backend:** `dsr/AttendanceController` in Sales.API
**API endpoints** (check controller methods and match pattern):
- Grid listing attendance records
- Create/edit attendance entry
- Fields: dsr_user_id, date, check_in_time, check_out_time, status (Present/Absent/Late)

### 2b: DSR Collection Page

**Route:** `/sales/dsr/collection`
**Backend:** `dsr/CollectionController`
- Grid listing collections
- Create collection entry
- Fields: dsr_user_id, stakeholder_id (customer), amount, collection_date, payment_method, reference

### 2c: DSR Daily Sales Page

**Route:** `/sales/dsr/daily-sales`
**Backend:** `dsr/DailySalesController`
- Grid with daily sales records
- Create sale with detail lines (product_id, qty, price)
- Total amount auto-calculated

### 2d: DSR Expense Page

**Route:** `/sales/dsr/expense`
**Backend:** `dsr/ExpenseController`
- Grid listing expenses
- Create expense: expense_type, amount, date, remarks, receipt_image

### 2e: DSR Target Page

**Route:** `/sales/dsr/target`
**Backend:** `dsr/TargetController`
- Grid listing sales targets
- Create target: dsr_user_id, target_amount, period (monthly/quarterly), start/end_date

### 2f: DSR Visit Page

**Route:** `/sales/dsr/visit`
**Backend:** `dsr/VisitController` (different from VisitPlan ‚Äî this is actual visit check-in/out)
- Grid listing visits
- Create visit: plan_detail_id, stakeholder_id, check_in_time, check_out_time, notes, visit_status

## Task 3: Security Misc Pages

### 3a: Audit Log Page

**Route:** `/master-settings/security/audit-log`
**Backend:** `AuditLogController`
- Read-only grid showing audit trail
- Columns: table_name, action (INSERT/UPDATE/DELETE), record_id, old_values, new_values, changed_by, changed_at

**Route:** `/master-settings/security/financial-year`
**Backend:** `FinancialYearController`
- Grid listing financial years
- Create/edit: name, start_date, end_date, is_closed
- Only one year can be active at a time

### 3c: Global Settings Page

**Route:** `/settings/global`
**Backend:** `GlobalSettingController`
- Key-value pair management
- Grid: key, value, description, category
- Modal for edit

### 3d: Notifications Page

**Route:** `/support/notifications`
**Backend:** `NotificationController`
- Grid listing notifications
- Columns: title, message, type, is_read, created_at, expiry_date
- Mark as read, delete
